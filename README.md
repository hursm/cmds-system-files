## Overview

CMDS (커맨드스페이스) is a comprehensive Personal Knowledge Management (PKM) system built on Obsidian. This repository contains the 5 core system files that provide complete guidance for AI agents and human users.

## Why This Structure?

In the AI era, your knowledge system needs clear documentation for both humans and AI agents to work together effectively.

### 🤖 AI Agents Need Context

When you use AI tools like Claude Code, ChatGPT, or Cursor, they don't know your folder structure, naming conventions, or workflows. These system files give AI the context it needs to help you effectively without breaking your existing patterns.

### 🧠 Humans Need Reference

Even the creator of a knowledge system can forget conventions over time. These files serve as living documentation - a single source of truth for how your system works, what goes where, and why decisions were made.

### 🔄 Human-AI Collaboration

The 5-file structure creates a bridge between human understanding and AI capabilities:
- **AI documents** (CLAUDE.md, AGENTS.md, CMDS.md) are loaded into AI context windows
- **Human documents** (Guide, HQ) provide quick reference for daily use

### 📈 Systems Scale Better

A 5,000+ note knowledge base without documentation becomes chaos. With clear system files, you can maintain consistency across years of notes and evolve your system without losing its foundation.

## Core Files Explained

### For AI Agents (loaded into context window)

| File | What It Does | When AI Uses It |
|------|--------------|-----------------|
| **CLAUDE.md** | Technical implementation guide specifically for Claude Code. Contains file operation rules, Obsidian syntax, vault commands, and indentation rules. | When Claude Code writes or modifies files in your vault |
| **AGENTS.md** | Portable technical guide for other AI coding agents (Gemini CLI, Codex, Cursor, Windsurf). Simpler structure without Claude-specific content. | When using non-Claude AI tools |
| **CMDS.md** | The "story" behind the system. User profile, system philosophy, detailed category explanations, workflow patterns. Helps AI understand WHY things are organized this way. | When AI needs context about your work, background, or decision-making |

### For Humans (referenced in Obsidian)

| File | What It Does | When You Use It |
|------|--------------|-----------------|
| **CMDS Guide.md** | Operational standards. Properties templates, naming conventions, folder structure, note types. The rulebook for daily note-taking. | When creating new notes, checking standards, or onboarding |
| **CMDS Head Quarter.md** | Navigation hub. Quick links to all 91 subcategories, ChatGPT GPT links, CMDS Process overview. Your home base. | When you need to find something fast |

## CMDS Architecture

### 9 Categories (100-900 Series)

| Category | Name | Purpose | Example Content |
|----------|------|---------|-----------------|
| 100 | Themes | Discovery & Connection | Interests, topics, variables, terminologies |
| 200 | Literature | Integration & Theory | Concepts, frameworks, theories, book notes |
| 300 | Data | Collection & Management | Surveys, databases, panel data |
| 400 | Methodologies | Analysis & Implementation | Research methods, statistics, ML, prompts |
| 500 | Products | Tools & Platforms | Obsidian, ChatGPT, Claude, n8n |
| 600 | Specialties | Expertise & Application | Knowledge management, Second Brain, Gen AI |
| 700 | Creatives | Expression & Content | YouTube, SNS, music, digital art |
| 800 | Outputs | Publication & Delivery | PhD, articles, lectures, consulting |
| 900 | Divisions | Operations & Management | 9 professional divisions |

### CMDS Process

```
Connect → Merge → Develop → Share
```

- **Connect** (100): Discover ideas by connecting interests and topics
- **Merge** (200): Build theoretical foundations by integrating knowledge
- **Develop** (300-600): Collect data, apply methodologies, use tools, build expertise
- **Share** (700-800): Create content, publish research, teach, consult
